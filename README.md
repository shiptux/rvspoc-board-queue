# RVSPOC 2026 真机预约使用指南

本仓库是 RVSPOC 真实 RISC-V 开发板(A210)的**时段预约入口**。
你在对应板子的预约 Issue 里评论 `/claim` 申请时段,到点后通过 SSH 接入板子做验证 / 调优。

> 板子是稀缺资源,请把它当作**验证和性能测试**环境,日常开发尽量在 QEMU 上完成。
>
> **当前状态**:**A210** 真机已开放(**2 台在线**),按板子分别预约,规则相同。

## 0. 前置条件(只需做一次)

1. **你的 GitHub 账号必须在组委会选手名册内**,否则 `/claim` 会被直接拒绝。如果你还不在名册:**队员**请让本队**队长**在[队员登记 Issue(#3)](issues/3) 里登记你的 GitHub 账号;**队长**若不在名册,邮件联系组委会 rvspoc@kubuds.cn。
2. **在 GitHub 账号添加你的 SSH 公钥** —— 这是接入的**唯一前置**,**没加到点就连不上**。具体步骤、原理、以及"实在加不了怎么办",见下方「[SSH 公钥(必读必做)](#ssh-公钥必读必做)」一节。

## SSH 公钥(必读必做)

### 为什么

到你的时段开始时,bot 会**自动**从 `https://github.com/<你的账号>.keys` 取你的**公钥**,装到板子上授权你登录。所以:

- 你**必须**事先在 GitHub 账号里加过 SSH 公钥,否则到点 bot 取不到 → **无法接入**,白白浪费时段。
- 全程只用**公钥**(可公开,只能验证身份、不能反推私钥);你的**私钥永远留在自己电脑**,不会上传给任何人、任何机器。

### 怎么加(三步)

1. 本地生成密钥对(已有可跳过):
   ```
   ssh-keygen -t ed25519 -C "rvspoc"
   ```
   一路回车,得到 `~/.ssh/id_ed25519`(私钥,**保密**)和 `~/.ssh/id_ed25519.pub`(公钥)。
2. 查看公钥内容:
   ```
   cat ~/.ssh/id_ed25519.pub
   ```
3. 加到 GitHub:**Settings → SSH and GPG keys → New SSH key**,把上面 `ssh-ed25519 ...` 一整行粘进去,保存。

**自检**:浏览器打开 `https://github.com/<你的账号>.keys`,能看到刚加的那行公钥,就成了。

### 隐私说明

加好后 `github.com/<你的账号>.keys` 是任何人可见的 —— 这是 GitHub 的正常设计,公钥本就用于公开分发,**安全**(拿到公钥推不出私钥)。若介意,可**专门为比赛生成一对新密钥**(不复用日常那把),公开它的公钥无妨。

### 实在加不了怎么办

极少数情况(账号策略限制等)无法在 GitHub 添加公钥时:

- 把你的**公钥**(`id_ed25519.pub` 的内容,以 `ssh-ed25519` / `ssh-rsa` 开头那行)**发邮件**到组委会邮箱 **rvspoc@kubuds.cn**,并注明你的 **GitHub 账号**。
- 组委会会**手动授权**(特殊处理),之后你照常 `/claim` 接入。

> **只发公钥,别发私钥**:带 `.pub` 的(`id_ed25519.pub`)才是公钥;`id_ed25519`(没 `.pub`)是**私钥**,绝不能外发 —— 任何人索要私钥都是诈骗。
> 这里特意走**邮件**而不是公开贴出,就是为了万一你发错,也只到组委会手里、不会当众泄露。

## 1. 怎么预约

在板子对应的预约 Issue 里**评论**一条:

```
/claim YYYY-MM-DD HH:MM Nh
```

- 时间为**北京时间(UTC+8)**
- `Nh` 是时长,例如 `2h`、`72h`(小数也行,如 `0.5h`)
- 例:`/claim 2026-06-15 19:00 4h`

| 机器类型 | 预约入口 | 状态 |
|---------|---------|------|
| A210(board-01) | 本仓库 Issue #1 | 开放预约 |
| A210(board-02) | 本仓库 Issue #2 | 开放预约 |

> 机器**不绑题**,任意赛题都可用 A210 做验证 / 调优。两台配置相同,哪台空闲约哪台;同一时间一人只能占一台。

bot 通常 1 分钟内回复:
- `[OK] slot 已排定` → 成功,附起始时间
- `[FAIL] ...` → 失败,附原因(未注册 / 不在服务窗口 / 超时长上限 / 时间冲突 / 同时占两块板 等)

到点时 bot 会自动开通你的账号,并在 Issue 回复"slot 已开始"。

## 2. 规则

- **服务窗口**:周一 ~ 周六全天可申请新时段。**周日不接新申请**(留作维护 / 批量重置)。已开始的时段可以跨周日继续。
- **单次最长 72 小时**,**每人每周累计上限 72 小时**(可能按实际竞争情况调整)。
- **同一时间一个人只能占一块板**(不能同时预约两块)。
- **同一块板同一时段只能一人**,先到先得。
- **最多提前 7 天预约**(滚动窗口):不能一次性占用更远的时段,保证大家都有机会;过几天日历滚动开了再约即可。
- 在以上限制内(7 天窗口、服务窗口、不冲突、不超时长 / 每周上限),预约哪天都行。

## 3. 怎么连接

把下面这段加进你本地的 `~/.ssh/config`(把 `<你的GitHub账号>` 和 `<board>` 替换成实际值):

```
Host rvspoc-bastion
    HostName 8.159.139.62
    User <你的GitHub账号>
    IdentityFile ~/.ssh/id_ed25519
    IdentitiesOnly yes

Host rvspoc-lab
    HostName localhost
    Port 2222
    User <你的GitHub账号>
    IdentityFile ~/.ssh/id_ed25519
    IdentitiesOnly yes
    ProxyJump rvspoc-bastion

Host rvspoc-board
    HostName <board>
    User <你的GitHub账号>
    IdentityFile ~/.ssh/id_ed25519
    ProxyJump rvspoc-lab
```

- `<board>` 填你预约的板子:`board-01` 或 `board-02`
- `IdentityFile` 指向你的**私钥**(其公钥已加到你的 GitHub 账号)
- 然后日常就一条命令:

```
ssh rvspoc-board
```

传文件:

```
scp -O ./mywork.tar rvspoc-board:~/
```

> 注意:你在跳板机(bastion)和控制机(lab-control)上**拿不到 shell**,它们只用于透传。
> 这是正常的 —— 你只能在板子上拿到 shell。

## 4. 在板子上

- 你的账号**有 `sudo` 权限**(免密),可以装运行依赖、跑 `perf`、改 sysctl 等。
- **长任务请用 `tmux`**(或 `mosh`),避免 SSH 断线丢进程:
  ```
  tmux new -s work
  # ...编译/跑测试...
  # 断开后重连: ssh rvspoc-board 然后 tmux attach -t work
  ```
- 板子是较精简的 Debian 系统,**编译工具链需要自己装**(目前预装的有 `python3` / `perf` / `gdb`,**没有 `gcc`/`make`/`cmake`/`git` 等**)。你有免密 `sudo`,直接装即可:
  ```
  sudo apt update && sudo apt install -y build-essential cmake git ninja-build pkg-config
  ```
- **任期结束板子会重置(见 §5),你装的东西会被清掉** —— 下个任期需重装。建议把环境搭建写成一个脚本,每次开始 slot 跑一遍最省事。
- 后续版本镜像会预装常用工具链,届时另行公告。

### 缺包怎么办

- 系统级包:`sudo apt install <包>`(你有免密 sudo)。
- 语言级:`pip install --user`、`cargo install` 等装到 home。
- 有希望进下一版镜像的常用包,可在预约 Issue 留言建议。

---

## 5. 重要:任期结束板子会被**完全重置**

- 你的时段结束后,板子会**擦除你的所有改动并重启回到初始状态**(下一个选手看不到你的任何数据,你也看不到上一个的)。
- **请在时段结束前保存好成果**:`scp` 回本地,或 `git push` 到你自己的仓库。
- 结束后你的账号和数据一并清除,**无法找回**。

## 6. 行为规范(违反会被取消资格)

- 不要尝试攻击 / 扫描其他主机或 IP(网络是隔离的,所有操作有审计记录)
- 不要挖矿、当代理跳板、发起对外攻击 / 大流量
- 不要破坏板子启动链(`/boot`、bootloader、分区表)—— 可能导致板子无法恢复
- 正常做移植 / 调优无需担心,放手用即可

风险声明:真机环境为公平比赛提供,请合理使用。异常行为触发审计告警后,组委会有权立即回收并取消资格。
